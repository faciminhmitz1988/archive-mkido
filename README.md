# archive-mkido

## 项目简介

`archive-mkido` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态 HTML 文件的集中存放与展示空间，方便对各类页面进行记录、整理与回溯。

## 目录结构

```
archive-mkido/
├── pages/          # 存放各独立 HTML 页面
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/         # 公共资源文件（如有）
│   ├── css/
│   ├── js/
│   └── images/
├── index.html      # 仓库首页 / 页面索引
└── README.md       # 本文件
```

- `pages/`：核心目录，每个 HTML 文件为一个独立归档页面。
- `assets/`：存放页面可能依赖的公共样式、脚本或图片资源。
- `index.html`：可选的入口页面，用于快速导航到各归档页。

## 页面归档说明

- 每个 HTML 文件均为独立完整的页面，可直接在浏览器中打开查看。
- 页面内容来源于不同时期的归档需求，可能包含信息展示、工具页面、静态快照等。
- 页面之间无依赖关系，可单独维护和更新。
- 文件名采用 `page-xxx.html` 的格式，便于按编号检索。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并按编号命名。
- 更新页面：直接修改对应 HTML 文件内容，注意保持文件独立性。
- 删除页面：移除对应文件，并建议在 `index.html` 中同步移除链接。
- 资源文件：若页面需要外部资源，请统一放置在 `assets/` 目录下，避免使用外部链接。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/archive-mkido.git
   ```
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件，或通过 `index.html` 导航查看。
3. 也可将仓库部署到任何静态托管服务（如 GitHub Pages），以在线方式浏览归档页面。
